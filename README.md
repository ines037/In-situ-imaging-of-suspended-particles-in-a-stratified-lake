# In-situ-imaging-of-suspended-particles-in-a-stratified-lake
# LISST-Holo2 Processing Pipeline

## Overview

This repository contains MATLAB scripts used to process and analyze data acquired with the LISST-Holo2 instrument during lake monitoring campaigns.

The workflow starts from raw holograms acquired in the field and produces particle size distributions, depth-resolved particle analyses, turbidity comparisons, and graphical outputs.

The processing chain combines:

* Raw hologram selection
* Hologram reconstruction using HoloBatch
* Formatting and correction of HoloBatch outputs
* Particle size distribution analysis
* Particle size versus depth analysis

# Data Processing Workflow

## Step 1 – Raw Data Acquisition

Raw LISST-Holo2 holograms are stored in:

```text
01_Raw data/YYYYMMDD/
```

Each campaign generally contains:

* Raw holograms acquired continuously during the cast
* CTD profile measurements

During deployment, the instrument is usually:

* stopped for approximately 20 seconds every 5 meters
* stopped for approximately 2 minutes within the turbidity peak

Only holograms acquired while the instrument is stationary are used for further analyses.

---

## Step 2 – Selection of Relevant Holograms

### FindRelevantRawHolograms.m

This function identifies holograms suitable for analysis.

The selected holograms must satisfy the following conditions:

* Instrument underwater (depth > 1 m)
* Instrument descending
* Instrument stationary at a sampling depth

The function removes:

* Surface images
* Images acquired during vertical movement
* Images acquired during ascent
* Short pauses that are not intentional sampling stops

Output:

* Indexes of holograms to keep for reconstruction and analysis

---

### Separate2profiles.m

Some deployments contain two independent casts in the same raw dataset.

This function:

* Detects the interruption between profiles
* Separates Profile 1 and Profile 2
* Produces a quality-control depth profile figure

Output:

* profile1_idx
* profile2_idx

---

## Step 3 – Hologram Reconstruction with HoloBatch

The selected holograms are processed using HoloBatch.

Outputs are saved in:

```text
03_Processed data/YYYYMMDD/
```

### Reconstructed Images

```text
03_Processed data/YYYYMMDD/Reconstructed images/
```

Contains reconstructed particle images generated by HoloBatch.

---

### Size Distributions

```text
03_Processed data/YYYYMMDD/Size distributions/
```

Contains particle statistics generated by HoloBatch.

#### ALL File

Example:

```text
045-0079_045-1197_All.csv
```

There is only one ALL file per acquisition date.

The ALL file contains:

* Date and time
* Depth
* Instrument parameters
* Number of particles
* Total particle volume
* Particle size distributions

One row corresponds to one hologram.

---

#### PSTAT Files

Examples:

```text
045-0079-pstat.csv
045-0080-pstat.csv
045-0081-pstat.csv
...
```

There are many PSTAT files per acquisition date.

One PSTAT file corresponds to one hologram.

They contain detailed particle measurements extracted from reconstructed images.

---

## Step 4 – Formatting HoloBatch Outputs

The files generated by HoloBatch require formatting before scientific analyses.

### CSVtitles.m

Adds variable names and units to the ALL file.

Purpose:

* Create readable column headers
* Add measurement units
* Standardize ALL file structure

Input:

```text
*_All.csv
```

Output:

```text
Modified *_All.csv
```

---

### ChangeDate_safe.m

Corrects acquisition dates stored in the ALL file.

Purpose:

* Extract campaign date from the folder name (YYYYMMDD)
* Update date columns
* Preserve the original file structure

Modified fields:

* Year
* Month
* Day
* Hour
* Minute
* Second

Only data rows are modified.

Input:

```text
*_All.csv
```

Output:

```text
Corrected *_All.csv
```

---

### MergeCSV_2.m

Synchronizes metadata between the ALL file and PSTAT files.

Purpose:

* Read metadata from the ALL file
* Update every PSTAT file

Transferred variables:

* Date
* Time
* Depth
* Deployment ID
* Number of particles

Input:

```text
*_All.csv
*-pstat.csv
```

Output:

```text
Updated PSTAT files
```

## Step 5 – Scientific Analyses

### SizeDistribution.m

Main script for particle size distribution analysis.

This script:

* Reads ALL files
* Reads CTD turbidity profiles
* Calculates SPMVC
* Calculates PSD
* Computes mean distributions by depth
* Computes distributions within turbidity peaks
* Compares epilimnion and hypolimnion distributions
* Detects dominant particle-size modes
* Generates publication-quality figures

Outputs include:

* PSD plots
* SPMVC plots
* Turbidity correlations
* Depth-level analyses
* MAT files containing processed variables

Results are saved in:

```text
04_Data analysis/
```

---

### SizeParticle.m

Analyzes particle size classes as a function of depth.

The script:

* Reads particle size distributions from the ALL file
* Groups particles into size ranges
* Computes relative volumetric contributions
* Calculates D50 (median particle diameter)
* Produces depth-resolved visualizations

Size classes:

* 8–20 µm
* 20–60 µm
* 60–125 µm
* > 125 µm

Outputs:

* Stacked particle-size profiles
* D50 depth profiles
* Exported figures

Results are saved in:

```text
04_Data analysis/
```

# Recommended Processing Order

For a new acquisition campaign:

```text
1. FindRelevantRawHolograms.m

2. Separate2profiles.m
   (only if two profiles are present)

3. HoloBatch

4. CSVtitles.m

5. ChangeDate.m

6. MergeCSV_2.m

7. PlotSizeDistr.m

8. SizeParticle.m


### Pipeline
Raw data
   ↓
FindRelevantRawHolograms
   ↓
HoloBatch reconstruction
   ↓
CSVtitles → ChangeDate → MergeCSV
   ↓
Size & PSD analysis scripts

```
